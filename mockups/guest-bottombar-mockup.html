👆 Try: tap Pantry, then toggle "Proposed" above to see the redesigned gate.

Today, three of five destinations end in a wall for a guest: 📸 Scan (popup), 🧺 Pantry (full screen), 👤 Profile (full screen). 🎡 Decide and 📒 History work fully without an account.
  • Scan & Pantry today: only a single "Sign Up — It's Free" button. A returning user who already has an account has no quick way back to Sign In from here — they'd have to find Profile first.
  • Profile today: already does roughly what you're asking for — "Sign In" primary + "or create a free account" link.
  • Proposed: the same friendly template everywhere — a soft icon badge, one warm sentence about what they're missing, then both "Sign Up — It's Free" and "Already have an account? Sign In" buttons. Consistent, low-pressure, and never strands a returning user.
  • New — History, empty state: a brand-new guest who taps History before ever spinning sees the same plain "No meals yet — spin the wheel! 🎡" as everyone today. Proposed: for guests only, swap that empty state for the same cute gate — "Remember every meal" + Sign Up/Sign In. A guest who already has history keeps seeing it as normal (no change, no gate).
This only touches the three gate screens/popup plus History's empty state — Decide and History-with-content need no changes.
